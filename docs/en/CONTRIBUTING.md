# Contributing to bk-sops
[Report Issues](https://github.com/TencentBlueKing/bk-sops/issues) and [Pull requests](https://github.com/TencentBlueKing/bk-sops/pulls) are always welcomed. Please read the following before code contribution


## Issue Management
We use Github Issues to track public bugs and feature requests.


### Search for known Issues first 
Please search for existing or similar issues first to avoid reporting the same issue again.


### Create Issue
When creating an issue, please provide a detailed description, screenshot or a video clip to help us locate the problem.


###  Pull Requests

Everyone can make contributions to make SOPS more powerful. The code team will monitor all pull requests and perform corresponding code review and tests. Once the code passes the test, we will accept the PR. However, it will not be merged to master immediately.

Please do the following before completing a PR: 

1. Fork the repository and create your branch from the community edition branch `release_humming_bird`.
2. Please modify the corresponding document and comment after changing the code.
3. Add license and copyright statement in the created file.
4. Make sure the coding style is consistent.
5. Perform adequate tests.
6. Then, you may submit your code via a Pull Request to the `release_humming_bird` branch.
