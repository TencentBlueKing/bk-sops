# 流程分析维度设计文档

## 一、概述

本文档定义了BK-SOPS流程分析的5个核心维度，明确了每个维度的数据来源、依赖关系和实施优先级。这些分析维度旨在从多个角度全面分析流程模板和执行数据，为流程优化、质量提升和智能决策提供数据支持。

### 设计原则

1. **数据驱动**：所有分析维度基于实际的数据模型和字段
2. **分阶段实施**：区分基础分析（可直接实施）和高级分析（需要知识库）
3. **可扩展性**：设计支持后续扩展和增强
4. **实用性**：分析结果可直接用于业务决策和流程优化

### 数据源分类

- **现成数据**（✅ 可直接使用）：来自现有数据库表，无需知识库支持
- **知识库数据**（⚠️ 需要知识库）：需要知识库构建完成后才能使用

---

## 二、流程结构分析维度

### 2.1 维度说明

流程结构分析维度主要分析流程模板的结构特征，包括节点数量、网关类型、流程复杂度、执行模式等。通过结构分析，可以识别流程的设计模式、发现结构问题、评估流程复杂度。

### 2.2 数据来源

#### 主要数据源

1. **pipeline_tree**（从 `TaskTemplate.pipeline_template.data` 或 `CommonTemplate.pipeline_template.data` 获取）
   - `activities`：任务节点字典，包含所有标准插件节点和子流程节点
   - `gateways`：网关节点字典，包含并行网关、排他网关、汇聚网关
   - `flows`：顺序流字典，包含节点之间的连接关系
   - `start_event`：开始节点
   - `end_event`：结束节点
   - `constants`：全局变量字典（可选，用于分析变量使用）

2. **TemplateStatistics**（模板统计表）
   - `atom_total`：标准插件总数
   - `subprocess_total`：子流程总数
   - `gateways_total`：网关总数
   - `input_count`：输入变量数
   - `output_count`：输出变量数

#### 数据获取方式

```python
# 获取流程树结构
template = TaskTemplate.objects.get(id=template_id)
pipeline_tree = template.pipeline_tree  # 返回完整的pipeline_tree字典

# 获取模板统计信息
template_stats = TemplateStatistics.objects.get(template_id=template.pipeline_template.template_id)
```

### 2.3 依赖关系

- **基础分析**（✅ 可直接用现成数据）：
  - 节点数量统计
  - 网关类型统计
  - 流程深度计算
  - 分支数统计
  - 基础流程模式识别（串行、并行、混合）

- **高级分析**（⚠️ 需要知识库）：
  - 复杂流程模式识别（部署流程模式、监控流程模式、故障处理流程模式等）
  - 流程模式匹配和分类
  - 基于模式知识的流程优化建议

### 2.4 分析指标列表

#### 基础指标

1. **节点数量指标**
   - 总节点数（`atom_total + subprocess_total`）
   - 标准插件节点数（`atom_total`）
   - 子流程节点数（`subprocess_total`）
   - 网关节点数（`gateways_total`）

2. **流程复杂度指标**
   - 流程深度（从start_event到end_event的最长路径长度）
   - 分支数（排他网关的数量）
   - 并行分支数（并行网关的分支数量）
   - 循环结构数（识别循环网关）

3. **流程模式指标**
   - 串行节点比例（串行执行的节点占比）
   - 并行节点比例（并行执行的节点占比）
   - 网关使用率（使用网关的流程占比）

4. **变量使用指标**
   - 输入变量数（`input_count`）
   - 输出变量数（`output_count`）
   - 变量引用率（变量在节点中的使用率）

#### 高级指标（需要知识库）

1. **流程模式识别**
   - 流程模式类型（部署流程、监控流程、故障处理流程等）
   - 模式匹配度（与标准模式的相似度）
   - 模式变体识别

2. **结构优化建议**
   - 冗余节点识别
   - 并行化建议
   - 结构简化建议

### 2.5 实施优先级

- **优先级：P0**（高优先级，可立即实施）
- **实施阶段**：阶段五基础部分
- **依赖**：无，可直接基于pipeline_tree和TemplateStatistics实施

## 三、流程执行分析维度

### 3.1 维度说明

流程执行分析维度主要分析流程的执行情况，包括执行频率、成功率、耗时、趋势等。通过执行分析，可以识别高频流程、发现执行问题、评估流程效率、预测执行趋势。

### 3.2 数据来源

#### 主要数据源

1. **TaskflowStatistics**（任务实例统计表）
   - `template_id`：流程模板ID（用于关联模板）
   - `task_template_id`：任务模板ID
   - `instance_id`：流程实例ID
   - `create_time`：任务创建时间
   - `start_time`：任务启动时间
   - `finish_time`：任务结束时间
   - `elapsed_time`：执行耗时（秒）
   - `category`：任务分类
   - `project_id`：项目ID

2. **TaskflowExecutedNodeStatistics**（节点执行详情统计表）
   - `instance_id`：流程实例ID（关联到TaskflowStatistics）
   - `task_instance_id`：任务实例ID
   - `template_node_id`：模板节点ID
   - `node_id`：执行节点ID
   - `component_code`：组件代码
   - `started_time`：节点开始执行时间
   - `archived_time`：节点结束时间
   - `elapsed_time`：节点执行耗时（秒）
   - `status`：是否执行成功（Boolean）
   - `is_skip`：是否跳过
   - `is_retry`：是否重试

3. **TaskFlowInstance**（任务实例表，用于获取任务状态）
   - `status`：任务状态（CREATED, RUNNING, FAILED, SUSPENDED, REVOKED, FINISHED）
   - `current_flow`：当前流程状态

#### 数据获取方式

```python
# 获取模板的所有执行记录
taskflow_stats = TaskflowStatistics.objects.filter(
    template_id=template_id
).order_by('create_time')

# 获取节点的执行详情
node_stats = TaskflowExecutedNodeStatistics.objects.filter(
    template_id=template_id
)
```

### 3.3 依赖关系

- **全部分析**（✅ 可直接用现成数据）：
  - 执行频率分析
  - 成功率分析
  - 耗时分析
  - 执行趋势分析
  - 失败原因分析

- **无需知识库支持**：所有分析指标都可以直接基于执行数据计算

### 3.4 分析指标列表

#### 执行频率指标

1. **总体频率**
   - 总执行次数（按template_id分组统计）
   - 日均执行次数
   - 周均执行次数
   - 月均执行次数

2. **时间分布**
   - 按小时分布（识别执行高峰期）
   - 按天分布（识别执行高峰日）
   - 按周分布（识别执行高峰周）

3. **频率排名**
   - 高频流程Top N（执行次数最多的流程）
   - 低频流程识别（执行次数很少的流程）

#### 成功率指标

1. **总体成功率**
   - 成功率 = 成功次数 / 总执行次数
   - 失败率 = 失败次数 / 总执行次数
   - 跳过率 = 跳过次数 / 总执行次数

2. **成功率趋势**
   - 按时间维度（日/周/月）统计成功率趋势
   - 成功率变化率（环比、同比）

3. **失败原因分析**
   - 节点失败统计（按component_code统计失败次数）
   - 失败节点排名（失败次数最多的节点）
   - 失败时间分布（失败发生的时间段）

#### 耗时指标

1. **总体耗时**
   - 平均耗时（所有执行的平均耗时）
   - 中位数耗时
   - 最大耗时
   - 最小耗时
   - 耗时标准差

2. **耗时分布**
   - 耗时分布直方图（按时间段统计）
   - P50/P90/P95/P99耗时（百分位数）

3. **耗时趋势**
   - 按时间维度统计平均耗时趋势
   - 耗时变化率（环比、同比）

4. **节点耗时分析**
   - 节点平均耗时（按component_code统计）
   - 耗时最长节点Top N
   - 节点耗时占比（节点耗时/总耗时）

#### 执行趋势指标

1. **执行次数趋势**
   - 按日/周/月统计执行次数趋势
   - 执行次数增长率

2. **成功率趋势**
   - 按日/周/月统计成功率趋势
   - 成功率变化趋势

3. **耗时趋势**
   - 按日/周/月统计平均耗时趋势
   - 耗时变化趋势

### 3.5 实施优先级

- **优先级：P0**（高优先级，可立即实施）
- **实施阶段**：阶段五基础部分
- **依赖**：无，可直接基于TaskflowStatistics和TaskflowExecutedNodeStatistics实施

## 四、节点使用分析维度

### 4.1 维度说明

节点使用分析维度主要分析标准插件节点的使用情况，包括使用频率、成功率、耗时、组合模式等。通过节点分析，可以识别热门节点、发现节点问题、评估节点效率、发现节点组合规律。

### 4.2 数据来源

#### 主要数据源

1. **TemplateNodeStatistics**（模板节点引用统计表）
   - `component_code`：组件代码
   - `template_id`：流程模板ID
   - `task_template_id`：任务模板ID
   - `node_id`：节点ID
   - `category`：模板分类
   - `is_sub`：是否子流程引用
   - `version`：插件版本

2. **TaskflowExecutedNodeStatistics**（节点执行详情统计表）
   - `component_code`：组件代码
   - `instance_id`：流程实例ID
   - `template_node_id`：模板节点ID
   - `status`：是否执行成功
   - `elapsed_time`：执行耗时
   - `is_skip`：是否跳过
   - `is_retry`：是否重试

3. **pipeline_tree**（流程树结构）
   - `flows`：顺序流字典，用于分析节点之间的连接关系
   - `activities`：任务节点字典，用于分析节点在流程中的位置

#### 数据获取方式

```python
# 获取节点的模板引用统计
template_node_stats = TemplateNodeStatistics.objects.filter(
    component_code=component_code
)

# 获取节点的执行统计
executed_node_stats = TaskflowExecutedNodeStatistics.objects.filter(
    component_code=component_code
)

# 获取流程树中的节点连接关系
template = TaskTemplate.objects.get(id=template_id)
pipeline_tree = template.pipeline_tree
flows = pipeline_tree.get('flows', {})
```

### 4.3 依赖关系

- **基础分析**（✅ 可直接用现成数据）：
  - 节点使用频率统计
  - 节点成功率统计
  - 节点耗时统计
  - 相邻节点组合分析（基于flows分析）

- **高级分析**（⚠️ 需要知识库）：
  - 节点组合模式识别（如："脚本执行+结果判断"、"部署+验证"等）
  - 最佳节点组合推荐
  - 节点替代关系分析

### 4.4 分析指标列表

#### 基础指标

1. **节点使用频率指标**
   - 节点使用次数（按component_code统计在模板中的引用次数）
   - 节点使用率（使用该节点的模板数 / 总模板数）
   - 节点使用排名（使用次数最多的节点Top N）

2. **节点成功率指标**
   - 节点成功率（成功执行次数 / 总执行次数）
   - 节点失败率（失败执行次数 / 总执行次数）
   - 节点跳过率（跳过次数 / 总执行次数）
   - 节点重试率（重试次数 / 总执行次数）

3. **节点耗时指标**
   - 节点平均耗时（所有执行的平均耗时）
   - 节点中位数耗时
   - 节点最大/最小耗时
   - 节点耗时标准差

4. **节点组合指标**（基于flows分析）
   - 相邻节点组合（分析flows中的source和target）
   - 常见节点序列（识别常见的节点执行序列）
   - 节点前置节点统计（统计每个节点的前置节点）
   - 节点后置节点统计（统计每个节点的后置节点）

#### 高级指标（需要知识库）

1. **节点组合模式识别**
   - 识别标准节点组合模式（如："脚本执行+结果判断"）
   - 组合模式匹配度
   - 组合模式使用频率

2. **节点替代关系**
   - 识别功能相似的节点
   - 推荐节点替代方案

3. **节点优化建议**
   - 基于最佳实践的节点使用建议
   - 节点组合优化建议

### 4.5 实施优先级

- **优先级：P0**（高优先级，可立即实施基础部分）
- **实施阶段**：
  - 基础部分：阶段五基础部分
  - 高级部分：阶段五高级部分（需要知识库）
- **依赖**：
  - 基础分析：无，可直接实施
  - 高级分析：需要知识库中的节点组合模式知识

## 五、业务场景分析维度

### 5.1 维度说明

业务场景分析维度主要分析不同业务场景（category）下的流程使用情况，包括场景分布、使用频率、成功率、趋势等。通过场景分析，可以识别热门场景、发现场景问题、评估场景效果、预测场景趋势。

### 5.2 数据来源

#### 主要数据源

1. **TaskTemplate / CommonTemplate**（流程模板表）
   - `category`：模板分类（OpsTools, MonitorAlarm, ConfManage, DevTools, EnterpriseIT, OfficeApp, Other, Default）

2. **TaskflowStatistics**（任务实例统计表）
   - `category`：任务分类
   - `template_id`：模板ID
   - `create_time`：创建时间
   - `elapsed_time`：执行耗时
   - 其他执行相关字段

3. **TemplateStatistics**（模板统计表）
   - `category`：模板分类
   - `template_id`：模板ID

#### 数据获取方式

```python
# 按分类统计模板
templates_by_category = TaskTemplate.objects.values('category').annotate(
    count=Count('id')
)

# 按分类统计执行情况
executions_by_category = TaskflowStatistics.objects.values('category').annotate(
    count=Count('id'),
    avg_elapsed_time=Avg('elapsed_time')
)
```

### 5.3 依赖关系

- **全部分析**（✅ 可直接用现成数据）：
  - 场景分布分析
  - 场景使用频率分析
  - 场景成功率分析
  - 场景耗时分析
  - 场景趋势分析

- **无需知识库支持**：所有分析指标都可以直接基于category字段计算

### 5.4 分析指标列表

#### 场景分布指标

1. **模板分布**
   - 各场景模板数量
   - 各场景模板占比
   - 场景模板排名

2. **执行分布**
   - 各场景执行次数
   - 各场景执行占比
   - 场景执行排名

#### 场景使用频率指标

1. **总体频率**
   - 各场景日均执行次数
   - 各场景周均执行次数
   - 各场景月均执行次数

2. **频率趋势**
   - 各场景执行次数趋势（按时间维度）
   - 场景使用增长率

#### 场景成功率指标

1. **总体成功率**
   - 各场景成功率
   - 各场景失败率
   - 场景成功率排名

2. **成功率趋势**
   - 各场景成功率趋势（按时间维度）
   - 场景成功率变化率

#### 场景耗时指标

1. **总体耗时**
   - 各场景平均耗时
   - 各场景中位数耗时
   - 场景耗时排名

2. **耗时趋势**
   - 各场景耗时趋势（按时间维度）
   - 场景耗时变化率

#### 场景效果指标

1. **综合效果**
   - 场景效果评分（综合成功率、耗时、使用频率）
   - 场景效果排名

2. **场景对比**
   - 不同场景的效果对比
   - 场景优劣势分析

### 5.5 实施优先级

- **优先级：P0**（高优先级，可立即实施）
- **实施阶段**：阶段五基础部分
- **依赖**：无，可直接基于category字段实施

## 六、质量分析维度

### 6.1 维度说明

质量分析维度是综合性的分析维度，综合上述所有分析维度的结果，对流程质量进行综合评估，包括质量评分、优化建议、异常识别、最佳实践识别等。通过质量分析，可以全面评估流程质量、发现质量问题、提供优化建议、识别最佳实践。

### 6.2 数据来源

#### 主要数据源

1. **综合数据源**（综合上述所有数据源）
   - 流程结构数据（pipeline_tree, TemplateStatistics）
   - 流程执行数据（TaskflowStatistics, TaskflowExecutedNodeStatistics）
   - 节点使用数据（TemplateNodeStatistics）
   - 业务场景数据（category字段）

2. **知识库数据**（⚠️ 需要知识库支持）
   - 流程模式知识
   - 节点组合模式知识
   - 最佳实践知识
   - 异常模式知识

#### 数据获取方式

```python
# 综合多个数据源
template = TaskTemplate.objects.get(id=template_id)
template_stats = TemplateStatistics.objects.get(template_id=template.pipeline_template.template_id)
taskflow_stats = TaskflowStatistics.objects.filter(template_id=template.pipeline_template.template_id)
node_stats = TaskflowExecutedNodeStatistics.objects.filter(template_id=template.pipeline_template.template_id)

# 从知识库检索相关知识（需要知识库支持）
# knowledge_base.query(pattern="deployment_flow_pattern")
```

### 6.3 依赖关系

- **基础分析**（✅ 可直接用现成数据）：
  - 质量评分计算（基于成功率、耗时、复杂度等）
  - 基础异常识别（成功率异常低、耗时异常长等）
  - 基础优化建议（基于统计异常）

- **高级分析**（⚠️ 需要知识库）：
  - 基于最佳实践的优化建议
  - 最佳实践识别
  - 异常模式识别（基于模式对比）
  - 流程模式匹配和优化

### 6.4 分析指标列表

#### 基础指标

1. **质量评分指标**
   - 综合质量评分（综合成功率、耗时、复杂度等指标）
   - 质量评分组成（各维度得分）
   - 质量评分排名

2. **质量维度评分**
   - 执行质量评分（基于成功率、耗时）
   - 结构质量评分（基于复杂度、节点数）
   - 使用质量评分（基于使用频率、稳定性）

3. **基础异常识别**
   - 成功率异常低（低于阈值）
   - 耗时异常长（超过阈值）
   - 使用频率异常低（很少使用）
   - 复杂度异常高（节点数、深度等超过阈值）

4. **基础优化建议**
   - 基于统计异常的优化建议（如：成功率低建议检查节点配置）
   - 耗时优化建议（识别耗时最长的节点）
   - 结构优化建议（识别冗余节点、并行化机会）

#### 高级指标（需要知识库）

1. **基于最佳实践的优化建议**
   - 流程模式优化建议（对比最佳实践模式）
   - 节点组合优化建议（推荐最佳节点组合）
   - 结构优化建议（基于最佳实践的结构建议）

2. **最佳实践识别**
   - 识别符合最佳实践的流程
   - 最佳实践流程特征提取
   - 最佳实践流程推荐

3. **异常模式识别**
   - 识别偏离正常模式的流程
   - 异常模式特征分析
   - 异常模式预警

4. **流程模式匹配**
   - 流程模式分类
   - 模式匹配度评分
   - 模式优化建议

### 6.5 实施优先级

- **优先级：P1**（中优先级，基础部分可立即实施）
- **实施阶段**：
  - 基础部分：阶段五基础部分
  - 高级部分：阶段五高级部分（需要知识库）
- **依赖**：
  - 基础分析：依赖其他4个维度的基础分析结果
  - 高级分析：需要知识库中的最佳实践知识和模式知识

## 八、总结

本文档定义了5个核心分析维度，明确了每个维度的数据来源、依赖关系和实施优先级。通过分阶段实施策略，可以快速交付基础分析能力，然后在知识库构建完成后增强高级分析能力。

**关键要点**：
1. 4个维度（流程结构、流程执行、节点使用、业务场景）的基础分析可以立即实施
2. 质量分析的基础部分可以立即实施，高级部分需要知识库
3. 所有高级分析功能需要在知识库构建完成后实施
4. 建议优先实施P0优先级的基础分析，快速交付价值
